Dino Broker @VERSION@

This a binary release of the Dino Broker. This release should contain the following directories:

axis
BookPriceFinder
The Book Price Finder example. A simple example which demonstrates how to use Dino Broker.
BookPriceFinderRemote
The Book Price Finder: Remote example. An example which accesses a remote Dino Broker deployed as a web service.
doc
Contains the JavaDoc documentation for the Dino Broker.
lib
Contains dino.jar and the support libaries which it uses.
ontology-cache
Stores commonly accessed ontologies so that they can be accessed even if the URL of the ontology cannot be reached. Additional cached ontologies can be added here manually. An entry must also be manually added to the file service.idx in this folder which is an index of ontology files and their associated URIs.
owls-services
OWL-S service descriptions which the Dino Broker will search to find a service match. Additional service descriptions can be added here and the Dino Broekr will search them..
RestaurantSearch
The Restaurant Search example. A more complex example. Uses a Dino Broker to search for restaurants and display maps of their location using a number of services.

When the Dino Broker is run for the first time an additional directory called "repository" will be created. This directory contains extracted service information from the services in the "owls-service" directory. If you get unexpected match results, especially after modifying a service description, then try deleting this file so that Dino Broker rebuilds it.

Requirements

Dino Broker requires the Java Runtime Environment version 5 or greater.

Deploying Dino Broker to Axis

Dino Broker can be deployed as a web service is Axis. This makes the same broekr accesible to many clients.

To deploy Dino Broker to Axis, perform the following steps:

  1. If you don't have an application server installed then you need to install one. This is necessary as Axis runs inside an application server. A good choice is Apache Tomcat.
  2. If you don't already have an Axis installation, install Axis, using these instructions.
  3. You should now have an "axis" directory inside your application server installation. Inside you should have an "WEB-INF" directory which contains a "lib" directory. You must copy all the JAR files in the sub-directories of your Dino "lib" directory to this "lib" directory. All the JAR files must be copied to the top levelof the "lib" directory in the Axis installation (that is, they should not be in sub-directories).
  4. Copy the directories "owls-services" and "ontology-cache" from your dino installation directory to your application server. In Tomcat, copy these directories to the top level directory of your Tomcat installation. In other application servers, the location may vary. These directories are reached using the paths "../owls-services" and "../ontology-cache" so copy these directories to the appropiate location in your application server installation.
  5. If your application server is already running, it may be necessary to restart it or to restart the Axis application within your application server.
  6. If necessary, modify the "host" and "port" properties in "build.xml" in the "axis" directory of your installation to the host and port of your application server. By default these have the valuea "localhost" and "8080" so if you have a default Tomcat installation on the same machine where you installed the Dino distribution then you will not need to change these.
  7. To deploy the Dino Broker web service, first make sure the application server is running. Change to the "axis" directory of you Dino installation and run the command "ant". You will need to have Ant installed. Otherwise, see the Axis documentation on how to use the AdminClient directly.

Undeployment

To undeploy, run the command "ant undeploy" in the "axis" directory of the Dino installation or see the Axis documentation on the AdminClient..

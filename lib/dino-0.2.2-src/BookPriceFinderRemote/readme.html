Book Price Finder: Remote

This is a test application which uses a remote Dino Broker which is deployed as a web service. This example is the same as the Book Price Finder example but it uses a remote broker rather than a lcoal broker. The source code is found in the "src" folder.

To Run

you will first have to have deployed Dino Broker as an Axis web application (instructions). You will then need to modify the file "remoteBookFinder.properties" in the "RemoteBookPriceFinder" folder and set the values of the properties "broker-host", "broker-port" and "broker-file" to the lcoation of the Dino Broker web service. The default URL is "http://localhost:8080/axis/services/DinoBrokerService" so if the broker is deployed locally to Apache Tomcat on port 8080 then you will not need to cahnge anything. Once this is done, you can run the example using one of the methods below.

Easy way

Run remoteBookFinder.jar in the RemoteBookPriceFinder folder, for example by clicking or double clicking on it. This should work in many cases. If it doesn't then use one of the methods below.

Java command line

At the console / command prompt, navigate to the RemoteBookPriceFinder directory and then execute "java -jar remoteBookFinder.jar".

Windows

Execute "run.bat" in the RemoteBookPriceFinder folder.

Ant

At the console / command prompt, navigate to the RemoteBookPriceFinder directory and then execute "ant run".

To Recompile

You will need the latest version of Ant.

Execute "ant" to compile and create remoteBookFinder.jar. Execute "ant run" to run.

Short Overview of the Code

The classes are the same as the Book Price Finder example except that the class BookPriceFinder communicates with a remote Dino Broker rather than creating a local Dino Broker instance.
